# QClaw Mobile - 产品需求文档 (PRD)

> 作者：QClaw AI 助手  
> 版本：1.0  
> 日期：2026-03-26

---

## 1. 产品愿景

**QClaw Mobile** 是 QClaw 桌面版的移动端客户端，让用户在手机上也能享受完整的 AI 助手体验。

**核心理念**：不是另一个聊天机器人，而是 QClaw 的随身版 —— 技能、定时任务、记忆、工具调用，全部带走。

---

## 2. 用户画像

**主要用户**：已有 QClaw 桌面版的用户，希望在手机上随时与我交流。

**用户场景**：
- 通勤时用语音问我问题
- 外出时查看/管理定时任务
- 随时查看记忆和上下文
- 移动端使用 Skills

---

## 3. 功能需求

### 3.1 登录认证模块 (优先级 P0)

**目的**：保护 AI 对话安全，只有授权用户才能访问。

**功能**：
- 首次启动 → 显示"连接设置"引导页
- 输入 Gateway URL + Auth Token
- 保存到本地（加密存储）
- 登录状态持久化
- 退出登录（清除本地数据）

**技术实现**：
- Token 存储在 Android KeyStore
- HTTPS 推荐，HTTP 也支持（局域网场景）

### 3.2 聊天界面 (优先级 P0)

**目的**：核心对话体验，对标 ChatGPT Mobile。

**功能**：
- 消息列表（气泡式，左右区分）
- 文字输入框 + 发送按钮
- Markdown 渲染（代码高亮、链接、图片）
- AI 打字动画（流式响应）
- 消息时间戳
- 复制消息内容
- 清空对话

**交互细节**：
- 底部输入框，键盘自动弹出
- 新消息 → 自动滚动到底部
- 发送时按钮 loading 状态
- 空消息不可发送

### 3.3 语音输入 (优先级 P1)

**目的**：解放双手，语音直接转文字。

**功能**：
- 长按录音按钮 → 开始录音
- 松开 → 停止录音，自动转文字
- 权限申请引导（麦克风权限）

**技术**：Android SpeechRecognizer（系统自带）

### 3.4 技能中心 (优先级 P1)

**目的**：展示 QClaw 的 Skills 系统。

**功能**：
- 技能列表（卡片网格布局）
- 技能分类（搜索/写作/工具/娱乐）
- 技能搜索
- 点击技能 → 展示技能详情和使用说明
- 内置技能预设（小红书、天气、日历等）

**内置技能**：
| 技能 | 说明 |
|-----|------|
| 小红书 | 发布小红书内容 |
| 天气 | 查询天气 |
| 日历 | 管理日程 |
| 搜索 | 搜索工具 |
| 新闻 | 最新资讯 |

### 3.5 定时任务 (优先级 P1)

**目的**：管理 QClaw 的 Cron 定时任务。

**功能**：
- 任务列表（显示名称/时间/状态）
- 新建任务（任务名 + cron 表达式 + 描述）
- 编辑任务
- 删除任务
- 任务启用/禁用切换
- 下次执行时间展示

**Cron 预设**：
- 每天早上 8 点
- 每天晚上 8 点
- 每周一早上 9 点
- 自定义

### 3.6 记忆管理 (优先级 P2)

**目的**：查看和管理 AI 的记忆。

**功能**：
- 长期记忆列表（MEMORY.md 内容）
- 每日笔记列表（memory/*.md）
- 搜索记忆内容
- 查看单条记忆详情

### 3.7 设置页面 (优先级 P0)

**功能**：
- Gateway 连接配置（URL + Token）
- 主题切换（深色/浅色/跟随系统）
- 清除对话历史
- 清除缓存
- 关于页面（版本号）
- 退出登录

### 3.8 推送通知 (优先级 P3)

**目的**：定时任务完成后通知用户。

**功能**：
- FCM 推送配置
- 通知开关
- 通知音效

## 4. 技术架构

### 4.1 连接模式

**不走 AI 模型本地运行**，而是 **Gateway API Client** 模式：

```
手机 App ←→ QClaw Gateway API ←→ QClaw AI 大脑
```

**理由**：
- 手机性能有限，无法高效跑大模型
- 复用现有 AI 能力，用户已有 QClaw 授权
- 开发成本低，质量高

### 4.2 API 对接

通过 QClaw Gateway HTTP API：

| 功能 | API |
|-----|-----|
| 发送消息 | `POST /v1/chat/completions` |
| 查询会话 | `GET /sessions` |
| 管理 Cron | `GET/POST /cron` |
| 技能列表 | `GET /skills` |
| 上传文件 | `POST /upload` |

**认证**：Bearer Token（从设置页配置）

### 4.3 技术栈

- **框架**：React 18 + TypeScript
- **构建**：Vite + Capacitor
- **样式**：Tailwind CSS v4
- **状态**：Zustand
- **路由**：React Router v6
- **存储**：Capacitor Preferences（加密 Token）

## 5. UI/UX 设计规范

### 5.1 设计语言

**风格**：Modern AI Native  
**参考**：ChatGPT + Apple Messages + Notion

### 5.2 色彩

```
主色：     #667EEA → #764BA2 (渐变)
背景：     #0A0A14 (深黑)
卡片：     #12121E
边框：     #1E1E30
文字主：   #FFFFFF
文字次：   #8888AA
成功：     #10B981
警告：     #F59E0B
错误：     #EF4444
```

### 5.3 字体

- 主字体：系统默认（中文用系统字体）
- 代码：Monospace

### 5.4 组件规范

- 卡片：圆角 16px，轻微阴影
- 按钮：圆角 12px，主色渐变
- 输入框：圆角 12px，深色背景
- 底部 Tab：4 个图标 + 标签
