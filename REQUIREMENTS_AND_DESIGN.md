## 第一部分：产品需求分析

### 1.1 产品定位

**QClaw Mobile** 是桌面端 QClaw 的完整移动版本，让用户在手机上拥有与桌面端**完全相同**的功能体验。

**核心理念：**
- 不是简化版，而是**完整复刻**
- 所有功能、所有通道、所有 Skills 都要支持
- 随时随地使用完整的 AI 助手

### 1.2 用户场景

| 场景 | 描述 |
|-----|------|
| 远程办公 | 出差在外，通过手机远程操控电脑任务 |
| 日常助手 | 随时随地与 AI 对话、查询信息 |
| 创意工作 | 素材搜集、内容创作、文档处理 |
| 开发工作 | 代码生成、GitHub 操作、项目管理 |
| 学习研究 | 论文查询、知识整理、学术辅助 |

### 1.3 核心功能模块

#### 1.3.1 聊天系统（双模式）

**对话模式**
- 传统聊天界面
- 消息收发（文字、图片、语音、文件）
- 流式响应展示
- 消息历史管理
- Markdown 渲染 + 代码高亮

**工作室模式**
- 拟人化小龙虾动画（Lottie）
- 任务状态实时展示
- 小龙虾动作随任务变化
- 浮窗聊天功能
- 工作室背景场景

#### 1.3.2 Agent 功能
- Pi Agent 集成
- 工具调用展示
- 思考过程（Thinking）
- 推理模式（Reasoning）
- 代码执行
- 网络搜索

#### 1.3.3 Skills 系统
- 技能市场浏览
- 技能安装/卸载
- 技能启用/禁用
- 技能配置管理
- 60+ 内置技能支持

#### 1.3.4 定时任务（Cron）
- 任务创建/编辑/删除
- Cron 表达式支持
- 任务执行历史
- 多种投递方式（消息/Webhook/内部）

#### 1.3.5 记忆系统
- 长期记忆（MEMORY.md）
- 每日笔记（memory/YYYY-MM-DD.md）
- 记忆搜索
- 自动压缩

#### 1.3.6 多通道支持
- 微信（WeChat Access）
- QQ（QQBot）
- Telegram
- Discord
- Slack
- iMessage
- 企业微信、飞书、钉钉等

#### 1.3.7 配置管理
- 配置文件导入/编辑
- **LLM 模型切换**
  - 默认大模型（GLM，每天 4000W token 免费额度）
  - OpenAI（需自配 API Key）
  - Claude（需自配 API Key）
  - 其他模型支持
- **Token 额度管理**
  - 显示今日已用 token 数
  - 显示剩余免费额度
  - 额度用尽提醒
  - 自动切换到付费模型
- API Key 管理
- 通道配置
- 插件配置

#### 1.3.8 媒体支持
- 图片上传/下载
- 语音转文字（STT）
- 文字转语音（TTS）
- 文件管理
- 视频支持

#### 1.3.9 高级功能
- 多代理路由
- 会话管理
- 消息压缩
- 权限控制
- 审计日志

### 3.2 关键技术决策

| 决策 | 方案 | 理由 |
|-----|------|------|
| 移动框架 | Capacitor | 支持 Node.js，可运行完整 Gateway |
| UI 框架 | React 18 | 跨平台，生态成熟 |
| 样式 | Tailwind CSS v4 | 响应式，开发快速 |
| 状态管理 | Zustand | 轻量级，易于使用 |
| 动画 | Lottie | 轻量级，支持复杂动画 |
| 数据库 | SQLite | 轻量级，无需服务器 |
| 后端 | OpenClaw | 完整功能，本地运行 |

### 3.3 风险评估

| 风险 | 影响 | 缓解方案 |
|-----|------|---------|
| Node.js 在 Android 上的性能 | 高 | 优化依赖，精简 Gateway |
| APK 体积过大 | 中 | 分包，动态加载 |
| 电池消耗 | 中 | 后台任务优化，定时任务调度 |
| 内存不足 | 中 | 内存管理，垃圾回收优化 |
| 网络连接不稳定 | 低 | 重试机制，离线缓存 |
